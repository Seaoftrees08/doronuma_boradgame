# doronuma_game_system.md

## 1. 全体構成と使用技術
インターネット上の画面と、裏側の審判役を分担させる構成とする。

* **画面の作成 (TypeScript + Next.js + TailwindCSS v4):** プレイヤーが見る画面、ボタン、カードの絵柄などを作るための道具。TailwindCSS v4 を使ってデザインを効率的に組み立てる。
* **ウェブサイトの置き場所 (Firebase Hosting):** 完成した画面をインターネット上に置き、アクセスできるようにする場所。
* **データの共有 (Cloud Firestore):** 「誰がカードを使ったか」「点数がどう変わったか」を全員の画面へ即座に伝えるリアルタイムの共有掲示板。
* **裏側の審判役 (Cloud Run):** 点数計算やカードを引く処理など、プレイヤー側で行うとズルができる作業を安全に行う仕組み。
* **プレイヤーの識別 (Firebase 匿名認証):** パスワード登録などをせず、ウェブサイトを開いた瞬間に見えない「参加券」を配り、画面を閉じても同じ人で復帰できるようにする仕組み。
* **自動更新 (Cloud Build + GitHub):** 新しい機能を追加した際に、自動で検査を行い、インターネット上のゲームを最新状態に書き換える仕組み。
* **時間管理システム（未来の予定表）:** 裏側の審判役の中に組み込まれる、自動で時間を管理する仕組み。

## 2. ロビー機能とゲーム開始の仕組み
あらかじめ用意した画面の枠組みの中に、「共有掲示板」から最新の設定データを読み込んで表示する。

* **部屋の作成と合流:**
    1.  主催者（ホスト）が部屋を作ると「合言葉」が発行される。
    2.  他の人は合言葉を入力して同じ待合室（ロビー）に入る。
* **参加判定（Ready機能）:**
    * ゲームに参加するプレイヤーは、ロビー画面で「Ready」にチェックを入れる。
    * チェックを入れていない人、またはゲーム開始後に新しく「ルームID」を入力して入ってきた人は、操作ができない「オブザーバー（見学者）」として参加する。
* **ルールの自由設定:**
    主催者はロビー画面で募集人数、山札の内訳、突然死カードを混ぜる範囲、自身のターン時間、割り込みの選択時間を変更できる。変更は即座に全員の画面に反映される。
    * **自身のターン時間:** 合成フェーズとドロー＆アクションフェーズを合わせた合計時間。
* **ゲーム開始:**
    人数が揃い、参加者がReadyになった後、主催者が「開始」ボタンを押す。その合図が掲示板に書き込まれると全員の画面が切り替わり、裏側の審判役が山札の準備を始める。

## 3. 自動進行と通信管理
* **ターンの流れ:**
    各プレイヤーのターンは次の順序で進む。
    1. **合成フェーズ:** +1点カードを1枚自動で受け取る。その後、任意で昇格合成を1回行える。合成の検証・処理は裏側の審判役（Cloud Run）が行う（詳細はセクション4参照）。
    2. **ドロー＆アクションフェーズ:** 4つの選択肢から1つを選んで実行する。
    3. **ターン終了:** 次のプレイヤーへ移る。

* **時間管理（自動予約システム）:**
    * 誰かの番が始まった瞬間、裏側の審判役は「未来の予定表」に「〇秒後にこのプレイヤーの時間切れ強制処理を実行する」という予約を書き込む。
    * 時間内にプレイヤーが行動した場合は、この予約を消去する。
    * 操作がないまま時間が経過すると、予定表が自動で審判役に命令を出し、強制的にその時点の状態に応じた処理を実行する。画面側の時計や通信状態に依存せず、確実に時間が管理される。
        * **合成フェーズ・ドロー＆アクションフェーズを含む自身のターン中に時間切れになった場合:** パスが選択される。手札に変化はない。

* **割り込みの処理:**
    妨害系カードが使われた場合のみ、割り込みの受付が始まる。「妨害への対抗」カテゴリのカードは「妨害系」カテゴリのカードに対してのみ有効であり、得点取得系や強奪系のカードには使用できない。攻撃が宣言されるとシステムが一時停止状態になり、カウントダウンが始まる。誰かが割り込むたびに時間は最初から数え直しになる。時間がゼロになった時点で、裏側の審判役が最後に出されたカードから順に効果を計算し、確定した結果だけを画面に送る。

* **通信の切断と見学者化:**
    * 2回連続で時間切れになったプレイヤーは「空席」扱いとなり、点数計算時に強制的に最下位（負け）となる。空席プレイヤーの番はパス処理が即座に行われ、順番が飛ばされる。空席プレイヤーを攻撃の対象にすることはできない。
    * 通信トラブルなどで画面を閉じてしまい、敗北が確定した後にゲームに復帰したプレイヤーは、「オブザーバー（見学者）」となる。画面を見るだけで操作はできない。ゲーム終了後にロビーに戻れば、再びプレイヤーとして登録・参加できる。

* **主催者の切断:**
    主催者の通信が切断された、または空席扱いになった場合、残りの参加者からランダムに新しい主催者が選ばれ、操作の権限が移る。

## 4. 勝利点カードの管理

勝利点カードは行動カードとは別に各プレイヤーが手持ちとして保持するカードである。枚数の上限はない。勝利点カードの移動・廃棄は、行動カードの効果によってのみ発生する。

### カードの種類と取得方法

| カード | 取得方法 |
|---|---|
| **+1点カード** | 合成フェーズ開始時に1枚自動付与。合成の素材としても使用する。 |
| **+3点カード** | 合成（昇格・小）によって生成される。 |
| **+5点カード** | 合成（昇格・大）によって生成される。 |
| **-3点カード** | 妨害系行動カードの効果によって付与される。自分自身で取得することはできない。 |

### 合成（昇格）

自分のターンの合成フェーズ中に、以下の合成を1回だけ行うことができる（任意）。合成の検証・処理は改ざん防止のため、すべて裏側の審判役（Cloud Run）が行う。

| 合成名 | 素材 | 生成物 |
|---|---|---|
| 昇格（小） | +1点カード × 2 | +3点カード × 1 |
| 昇格（大） | +3点カード × 2 ＋ -3点カード × 1 | +5点カード × 1 |

素材として消費されたカードは手持ちから取り除かれ、生成物が手持ちに加わる。素材が足りない場合は合成を実行できない。

## 5. 点数計算

ゲーム終了時、各プレイヤーの手持ち勝利点カードの合計値が最終得点となる。

| カード | 得点 |
|---|---|
| +1点カード 1枚 | +1点 |
| +3点カード 1枚 | +3点 |
| +5点カード 1枚 | +5点 |
| -3点カード 1枚 | -3点 |

* **空席（AFK）プレイヤー:** 手持ちカードの得点に関わらず、強制的に最下位（負け）となる。
* **同点の場合:** 順位は同じとする。
